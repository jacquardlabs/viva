| 1 | technical | The shared `_check_origin_and_length` helper sends its own error response and returns `None`, but a POST branch stores the result and falls through instead of returning — yielding a double-send (headers already written) or a body read on an already-failed request. | At merge, grep the three POST branches (`/submit`, `/next-round`, `/complete`) in server.py: each `None` result from the helper must be immediately followed by `return`. A branch that keeps processing after `None` is the bug. |
| 2 | technical | The new Origin check on `/complete` accidentally rejects the browser tab's own same-origin `/complete` call (if any browser path issues it), so the 2-second shutdown timer never starts and the session leaks anyway — reintroducing #112 through the fix meant to close it. | Verify the reused check still allows `http://127.0.0.1:<port>` and `http://localhost:<port>` origins (the exact set `/submit` already permits). Confirm the new qa-mode exit test observes the process actually terminating, driven by the same `/complete` call path. |
| 3 | product | The conditional `/complete` step in `brainstorming-qa.md` is prose the agent executes. If the standalone-vs-hand-off branch is worded as a footnote rather than an explicit if/else, a caller calls `/complete` unconditionally and kills the process a `/next-round` hand-off is about to reuse — the exact regression the design warns against. | Read the revised finish section: the two paths (standalone → call `/complete`; hand-off → skip, review round's own `/complete` finishes) must be a clear either/or a caller cannot collapse, not a caveat sentence. |
